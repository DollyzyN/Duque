O que é git;
Git é um sistema de controle de versão distribuído, ou seja, um software que permite acompanhar as alterações feitas em um projeto de desenvolvimento, como um código-fonte, ao longo do tempo. Ele foi criado por Linus Torvalds, o criador do Linux, e é amplamente utilizado por desenvolvedores e equipes de desenvolvimento de software. 

O que são esses comandos do git:
Os comandos Git são instruções que você usa no Git, um sistema de controle de versão distribuído, para gerenciar o histórico de alterações de um projeto de software. Eles são como as ferramentas que permitem que você rastreie, manipule e compartilhe o seu código com outros, garantindo que você tenha uma versão confiável do seu projeto

git status
git status é um comando fundamental do Git que mostra o estado atual do seu repositório, informando quais arquivos foram modificados, adicionados ou removidos, e quais estão na área de staging, prontos para serem comitados. Ele permite que você veja as alterações que estão pendentes e que ainda não foram incorporadas ao histórico do projeto. 

git commit
Em termos simples, um git commit é um instantâneo (snapshot) que o Git cria das alterações feitas nos arquivos de um projeto, marcando um ponto específico no histórico do projeto. É como "salvar" as modificações que você quer manter no registro do seu trabalho. 

git push
git push é um comando Git usado para enviar as alterações locais confirmadas (commits) do seu repositório local para um repositório remoto. Em outras palavras, ele permite que você compartilhe seu trabalho com outros colaboradores ou faça backup do seu código em um servidor remoto.
